Claude Code Frontend Design Skill

★ 推荐 Claude Code Plugin
来源：Anthropic / GitHub  ·  作者：Anthropic 官方  ·  链接：github.com/anthropics/claude-code
  • 这不是一个 dashboard 本身，而是 2026 最重要的 vibe-code 元工具——Anthropic 官方的前端设计技能插件，277,000+ installs，彻底改变了 AI 生成界面的默认美学。
  • 核心机制：编码前先声明「审美方向」（极简主义、编辑风格、奢华暗黑、复古未来等），不允许直接扑向 shadcn 默认样式。
  • 明确的字体禁令：Inter / Roboto / Arial / Space Grotesk 都被列为「陈词滥调」；推荐选择有个性的展示字体配精致正文字体。
  • 颜色禁令：白底紫渐变、过度平均分配的胆小调色板被标注为反模式；推荐主色 + 锐利点缀色的层次结构。
  • Ethan Mollick（沃顿教授）在 X 上评价："ends the purple gradient and Arial font tyranny"，并表示已用它重设计了所有 vibe-coded 应用。这条推文引爆了 2026 年初前端 AI 设计讨论。
审美评分 ★★★★★

Meta-find：本身没有可截图的 UI，但它是所有「好看的 vibe-coded dashboard」背后的底层规则引擎。 任何认真做 AI 生成界面的 FP&A 都应该了解这套约束体系。

Ignidash — Open-Source AI Finance Simulator

★ 推荐 Next.js + Azure OpenAI HN Show HN · Jan 2026
来源：Hacker News Show HN (Jan 20, 2026)  ·  作者：schelskedev  ·  链接：github.com/schelskedevco/ignidash
  • 开源 ProjectionLab 替代品——长期个人财务规划仪表盘，亮暗双主题，专为 FP&A 类用户设计的蒙特卡洛模拟 + 历史回测可视化。
  • 技术栈：Next.js 16 / React 19 / TypeScript / Tailwind CSS / Convex（实时数据库）/ Azure OpenAI，定价约 $5/月。
  • 设计亮点：亮色版采用纸白底色 + 深墨色文字 + 蓝绿色图表；暗色版近黑底 + 高对比度折线图。布局遵循 Stripe-style 左侧导航 + 右侧数据密集区结构。
  • AI 对话层嵌入财务预测界面——可直接问「如果我推迟退休 3 年，胜率提高多少」。这种 chat-over-data 模式是 2026 年 FP&A 工具的核心范式。
  • 美中不足：字体仍用 Tailwind 默认 sans（很可能是 Inter），缺乏 display serif 的质感，但整体数据密度与布局克制程度符合本栏审美基线。
审美评分 ★★★★☆
03 / 06

Vault Dashboard — Investment Admin Template

★ 推荐 Next.js 16 + shadcn/ui
来源：DashboardPack / UI8  ·  作者：Capricorn Build  ·  链接：dashboardpack.com/vault-dashboard
  • 这是 2026 年少数在字体选择上真正克制的投资仪表盘模板：正文使用 Plus Jakarta Sans（线条感优雅，远好于 Inter）+ 数字/代码区域用 Geist Mono（Vercel 自研等宽，现代感强）。
  • 色板：Robinhood-style 近白底 + 墨绿色涨跌标注 + 中性灰边框，完全回避了「科技蓝」和「紫渐变」套路。
  • 57+ 页面覆盖 Portfolio、Watchlist、Earnings Calendar、Dividends、Tax Center 等财务场景；125+ 静态路由，生产级完成度。
  • Storybook 组件库 + 15 个交互故事，图表懒加载 + IntersectionObserver，性能取向而非「酷炫动效」——这与 Stripe Dashboard 哲学高度一致。
  • Tailwind CSS v4 + React 19——当前最新技术栈，可作为 FP&A dashboard 的工程参考标准。
审美评分 ★★★★☆

注：此模板并非 AI 「一键生成」，而是 AI 工具辅助开发的精工模板。 代表了「好的 vibe-code 目标态」：工程与美学都到位。

04 / 06

VoltAgent / awesome-design-md

★ 推荐 Claude / Cursor 通用 GitHub Trending
来源：GitHub / VoltAgent  ·  作者：VoltAgent Team  ·  链接：github.com/VoltAgent/awesome-design-md
  • 69 个品牌级 DESIGN.md 文件库——把 Stripe、Revolut、Wise、Apple、Vercel、Linear 等顶级产品的视觉语言提炼为 AI 可读格式，一键注入项目即可生成品牌匹配 UI。
  • 金融/分析类品牌设计系统：Stripe、Revolut、Wise、Binance、Coinbase、Kraken、Mastercard、PostHog 均在列——这正是 FP&A dashboard 审美的参照系原型。
  • 开发工具设计系统：Vercel（冷白 + Geist）、Linear（极简暗黑 + 密集 Typography Hierarchy）、Cursor、Supabase——覆盖了用户偏好的「Linear 极简风」源头。
  • 使用方式极为克制：一个 Markdown 文件 → drop into CLAUDE.md → AI 生成时自动遵循设计语言，无需重复描述字体/颜色规则。
  • 与本栏调研直接可用：明天做 dashboard 时，把 stripe.md 或 linear.md drop 进去，可以直接生成符合审美基线的界面。
审美评分 ★★★★★

这是一个「元设计工具」而非单一作品，但对 FP&A 做 AI dashboard 的实用价值极高。 配合 Claude Code Frontend Design Skill，可以系统性杜绝「AI 生成糊感」。

Chris Lee — Vibe-Coded Wealth Dashboard

Claude Code + Plaid X / Twitter · Apr 2026
来源：X (@techcontrarian)  ·  作者：Chris Lee  ·  链接：x.com/techcontrarian/status/2036…
  • Claude 作为「财富顾问」——用 Claude Code vibe-coded 一个私人网站，通过 Plaid 接入近 10 年收入/支出历史 + 股票交易记录，AI 层做实时财务分析和建议。
  • 这正是 2026 年 FP&A dashboard 的新范式：不再是「展示数据的看板」，而是「可对话的财务顾问界面」——数据 + AI + 实时 API 三合一。
  • 类似开源实现（github.com/mkash25/Claude-powered-AI-native-financial-dashboard）：Next.js + Tailwind + Supabase + Python 数据管道，$5/月 Claude API 成本，跑 RSI/MACD/布林带 + 基本面 + 新闻聚合后生成 buy/hold/sell 建议。
  • 设计上：私人项目，无公开设计截图，但原推图片显示数据密集的暗色主界面 + 左侧账户导航 + 中央净资产折线图，整体克制专业。
  • 警示：「私人财务 dashboard」这类项目需要特别注意 Plaid API 权限范围与数据本地化——不适合团队共享或企业部署，但作为个人 FP&A 工具极具参考价值。
审美评分 ★★★☆☆

设计评分保守（无公开完整截图），但作为「AI 财务仪表盘」的产品形态参考价值极高。 Claude Code + Plaid 的组合值得追踪。

Revlytics — SaaS 财务分析仪表盘

Lovable Made with Lovable
来源：madewithlovable.com  ·  作者：Revlytics Team  ·  链接：revlytics.lovable.app
  • 用 Lovable 生成的完整 SaaS 财务分析平台，覆盖 MRR、ARR、客户 LTV、Churn Rate、Failed Payments、Revenue Forecast——财务侧指标全覆盖。
  • UI 亮点：KPI 卡片用绿/红趋势指示器 + 迷你 sparkline 图表（1行内显示趋势），12 个月 MRR 折线图 + 新增/流失用户堆叠柱状图——数据密度与可读性平衡得不错。
  • 颜色系统：浅灰底 + 墨色文字 + 绿色正向指标 / 红色负向指标，标准但不廉价——至少没有「科技蓝卡片阴影套娃」问题。
  • Lovable 生成的最大局限：字体仍是默认 sans-serif stack，缺乏品味；组件间距和 font-size hierarchy 略显机械。
  • 作为 FP&A 同类工具参考：这基本是「好用但不够好看」的水平——功能完整度 ★★★★★，视觉品味 ★★★。对比 Stripe Dashboard 还有明显差距。
审美评分 ★★★☆☆

代表了 2026 年 Lovable 生成界面的典型水平：功能强，但未经「前端设计 Skill」处理就还是偏通用。 对 FP&A 用户有直接功能参考价值。

本周趋势

第一周观察：AI 界面审美正在分化

2026 年 4 月，vibe-coded dashboard 正经历一场「审美分化」：一边是 Lovable / bolt.new 批量产出的「功能完整但视觉平庸」的 shadcn 默认风，另一边是 Claude Code Frontend Design Skill（277K installs）、VoltAgent DESIGN.md 库等「元工具」正在系统性地改变 AI 生成 UI 的默认美学。Ethan Mollick 称之为「终结紫色渐变与 Arial 字体独裁」——这不是夸张，而是一场真实发生的品味迭代。

对 FP&A 用户来说，最值得关注的是「对话层覆盖数据层」的产品范式——Chris Lee 的 Plaid + Claude 私人财富仪表盘、Ignidash 的 Monte Carlo + AI 问答、Revlytics 的 AI 预测建议，都指向同一个方向：管理层 dashboard 的下一步不是更漂亮的图表，而是可以直接问「这个 budget variance 怎么解释」的 AI 界面。这是小红书 FP&A 需要提前布局的产品形态。
